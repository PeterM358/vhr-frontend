# Frontend Audit

## Goal
Understand the real current state of the frontend before new changes.

## Need to identify
- Current app entry point
- Current navigation structure
- Current auth flow
- Owner flow screens
- Shop flow screens
- Public-access screens
- Unused screens/components/files
- Duplicate screens or old versions
- Broken imports or dead routes

## Rules
- Do not delete files immediately
- First mark candidates as:
  - active
  - maybe unused
  - unused confirmed
- Prefer small safe cleanup
