# Frontend Project Goals

## Main Product Goal
Make the app usable for both guests and authenticated users, while keeping the repair marketplace flow simple.

## Immediate Goal
Change the landing page so the app opens with a public home screen instead of the login screen.

## Guest User Goal
Guest users should be able to:
- open the app
- browse repair shops
- open shop details
- call repair shops
- view map / services if available

Guest users should NOT be able to:
- create repair requests
- add vehicles
- manage profile
- access private owner/shop features

Restricted actions should redirect guests to login/register.

## Authenticated Vehicle Owner Goal
Authenticated vehicle owners should be able to:
- browse shops
- add vehicles
- create repair requests
- log repairs
- request repairs
- manage their vehicles and repairs

## Authenticated Shop Goal
Authenticated shops should be able to:
- access shop dashboard
- manage repairs
- manage offers
- manage promotions
- manage clients
- use existing shop flow without disruption

## Product Direction
The app should feel useful even before registration.
Browsing should be public.
Creating and managing data should require authentication.
